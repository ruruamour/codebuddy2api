账号池

批量导入 ck Key

每行一种格式：ck_xxx 或 账号名,ck_xxx 或 账号名\tck_xxx。 默认会按下方参数创建账号；完整 Key 只写入服务端 SQLite，不会再从接口返回。

接入说明

        

          

            
管理 Key

            
Admin Key / 对外 API Key 仍建议保留在服务器 env 中管理，不在网页里直接改，避免误操作把自己锁出管理面。

            
这个页面管理的是 CodeBuddy 上游账号 ck Key、并发、代理、健康状态和消耗统计。

          

          

            
sub2api 接入

            
base_url = http://127.0.0.1:18182

            
model = glm-5.1 / minimax-m2.7 / kimi-k2.6

            
sub2api 继续按站内模型价格计费；这里记录上游真实 usage.credit 方便对账。

          

          

            
Credits 查询

            
目前未发现 CodeBuddy Key 形态公开稳定的余额查询接口。面板里的 credit 使用量来自每次上游响应尾包 usage.credit 累计，这是当前不会额外消耗额度且最稳定的口径。

            
如果后续确认了官方 credits/balance endpoint，可以接到这里作为“真实余额同步”。

          

        

      

      

        
危险操作说明

        
删除账号会从本服务 SQLite 删除该 ck Key 记录及其累计统计。建议先停用观察，再删除。

        
轮换 Key：在账号表格的“新 ck Key”输入框填新 key 后保存；留空表示不修改原 key。
